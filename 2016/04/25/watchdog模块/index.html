watchdog模块

发表于 2016-04-25   |   分类于 python   |     |   阅读次数

前言

watchdog是python的第三方库，利用操作系统的API来监控目标目录或者文件的变化， 在linux系统下，借用了inotify机制
参照observers/__init__.py

Class Platforms Note
Inotify Linux 2.6.13+ inotify(7) based observer
FSEvents Mac OS X FSEvents based observer
Kqueue Mac OS X and BSD with kqueue(2) kqueue(2) based observer
WinApi MS Windows Windows API-based observer
Polling Any fallback implementation

watchdog以及需要用的模块介绍

打开文件操作

name:

  • 文件名， 是open函数唯一强制参数

mode:

值 描述
‘r’ 读模式
‘w’ 写模式
‘a’ 追加模式
‘b’ 二进制模式(可以添加其他模式)
‘+’ 读/写模式(可以添加其他模式)

注: 在二进制模式下，不会转换如换行符等

buffering:

  • 0|False：I/O输入和输出是无缓冲的(读写操作直接针对磁盘)
  • 1|True：I/O有缓冲，数据改动存在内存中，flush或close后才会更新磁盘的数据
  • 大于1：代表缓冲区的大小

随机访问操作

offset:

  • 偏移量， 表示一个字节(字符)数

whence:

  • 默认为0：表示偏移量从文件开头开始计算的
  • 值为1：表示相对于当前位置移动(此时offset可以为负数)
  • 值为2：表示相对于文件结尾的移动

tell:

  • 该方法返回当前文件的位置

watchdog.events.FileSystemEventHandler

基本文件系统事件处理程序
可以重载几种方法:

  • on_any_event(event)：所有事务的处理程序
  • on_created(event)：文件或者目录被创建的时候调用
  • on_deleted(event)：文件或目录被删除的时候调用
  • on_modified(event)：文件或目录被修改的时候调用
  • on_moved(event)：文件或目录被移动或重命名时候调用

watchdog.observers.Observer

观察目录并给事件处理程序分配调用

watchdog案例

  • 文件被删除时调用close()
  • 文件被创建时获得self.fd和self.offset
  • 文件被新增时打印出新增的内容
  • 文件被重命名时先关闭原文件再打开新文件并赋值self.fd和self.offset
